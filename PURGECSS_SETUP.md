## Dynamic Class Names (Safelist)

### ŌØī Problem: Purging Dynamic Classes
If you generate class names dynamically (like the category badges), PurgeCSS might remove them because it can't find them in the code:

```php
$categoryName = strtolower($product['category']);
// Result: badge-category-produce (not found in source files, unsafe!)
```

### Ō£ģ Solution: Safelist
The updated `tailwind.config.js` includes all dynamic classes in the `safelist`:

```javascript
safelist: [
  'badge-category-produce',
  'badge-category-dairy',
  // ... etc
],
```

**When to add to safelist:**
- Theme color variants (`bg-category-*, text-category-*`)
- Status indicators (`is-open`, `is-scrolled`, `show`)
- Role-based classes (`admin-*, user-*`)
- Any class generated by PHP logic

## Troubleshooting

### ŌØō CSS Classes Disappearing in Production
**Cause:** New class names not in safelist or content paths

**Fix:**
1. Add dynamic classes to `safelist` in `tailwind.config.js`
2. Ensure file path in `content` array includes the file using the class
3. Rebuild: `npm run tailwind:build`

### ŌØō CSS File Too Large (Still 1+ MB)
**Cause:** Not running minified build

**Fix:**
```bash
npm run tailwind:build  # Always use --minify flag
```

### ŌØō Build Takes Too Long
**Cause:** Scanning too many files

**Fix:**
- Verify `content` paths are specific: `./src/Views/**/*.php` Ō£ģ
- Avoid `**/*` patterns that scan everything ŌØī

### ŌØō npm is not installed
**Fix:**
1. Download Node.js from nodejs.org
2. Run: `npm install` in project root
3. Then: `npm run tailwind:build`
